<objective>
Two iOS performance fixes:

1. **DateFormatter caching (REQ-PERF-07):** `BudgetViewModel` and `ReceiptView` each create
   `DateFormatter` instances inline — on every `load()` call and on every SwiftUI render
   respectively. Move all `DateFormatter` allocations to `private static let` properties.

2. **BudgetViewModel API fetch (REQ-PERF-08):** `BudgetViewModel.load()` currently calls
   `getTransactions(limit: 200)` and computes spend locally. Replace with a call to
   the new `GET /api/budget-summary` endpoint, which returns `{"monthly_spend": <float>}` directly.

Purpose: Eliminates repeated DateFormatter allocation (expensive Obj-C object) on hot paths,
and removes the 200-transaction payload transfer/parse just to display one number.

Output: Modified `BudgetViewModel.swift`, `ReceiptView.swift`, `APIEndpoints.swift`, `APIClient.swift`.
</objective>

Replace the inline `DateFormatter` allocation in `currentMonthPrefix()` with a
`private static let` on the class:

```swift
// Add as a class-level static property (before or after other properties):
private static let monthPrefixFormatter: DateFormatter = {
    let f = DateFormatter()

    f.locale = Locale(identifier: "en_US_POSIX")

    return BudgetViewModel.monthPrefixFormatter.string(from: Date())
```

Note: `DateFormatter` is NOT thread-safe for concurrent access, but since
`BudgetViewModel` is `@MainActor`, all accesses are on the main thread — a `static let`
is safe here.

```swift
func load() async {
    isLoading = true
    errorMessage = nil

    let apiClient = APIClient.shared   // use however the existing code references APIClient
    let result = await apiClient.fetchBudgetSummary()

    switch result {
    case .success(let summary):
        monthlySpend = summary.monthlySpend
    case .failure(let error):
        if case .unauthorized = error {
            authManager.handleUnauthorized()
            isLoading = false
            return
        }
        errorMessage = error.localizedDescription
    }

    isLoading = false
}
```
