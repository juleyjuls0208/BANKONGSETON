<objective>
Close two runtime bugs introduced in Phase 32 that prevent the iOS and Android budget summary features from working at runtime.

**Bug 1 — iOS URL double-prefix:** `APIEndpoints.budgetSummary = "/api/budget-summary"` but
`baseURL = "https://juley2823.pythonanywhere.com/api"` already ends in `/api`, so the runtime
URL becomes `/api/api/budget-summary` → 404 on every budget summary request.

**Bug 2 — Response shape mismatch (both platforms):** Backend `/api/budget-summary` returns
`{"monthly_spend": <float>}`. iOS `BudgetSummaryResponse` declares `{spent, limit, percent,
currency}` with no CodingKeys. Android `BudgetSummaryResponse` declares the same with no
@SerializedName. Result: `spent` always decodes as 0.0 on both platforms.

Purpose: Complete REQ-PERF-07 (iOS budget wiring) and REQ-PERF-09 (Android budget wiring),
both of which are blocked by these two bugs.

Output: Fixed iOS URL path, fixed iOS and Android response decoders, working budget summary
on both platforms.
</objective>
