<objective>
Add startup guards to admin_dashboard.py and web_app.py that abort if FINANCE_PASSWORD is unset or uses the known-insecure default "finance2025".

Purpose: The hardcoded default means any deployment that forgot to set the env var silently runs with a publicly-known password. The guard makes this a hard failure at startup, not a silent security hole in production.
Output: Two files updated with identical guard blocks; all inline defaults removed.
</objective>
