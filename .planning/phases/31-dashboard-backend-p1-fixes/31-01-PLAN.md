In `backend/resilience.py`, find the `WriteQueue` class (lines 126-234) and locate its `process_queue` method.

Replace the current unbounded `while not self.queue.empty()` loop with a snapshot-bounded loop:

```python
def process_queue(self):
    snapshot_size = self.queue.qsize()
    for _ in range(snapshot_size):
        if self.queue.empty():
            break
        item = self.queue.get()
        if not self._write(item):
            item['attempts'] = item.get('attempts', 0) + 1
            if item['attempts'] < 3:
                self.queue.put(item)    # re-queued for next call, not this one
            else:
                self.failed_queue.put(item)
                logger.error(
                    f"event=write_dropped attempts=3 "
                    f"item_type={item.get('type', 'unknown')} "
                    f"reason='max retries exceeded, item moved to failed_queue'"
                )
```

Key constraints:
- Use `snapshot_size = self.queue.qsize()` at the START of the call — this is the ceiling
- Items re-queued this call are NOT re-processed until the NEXT `process_queue()` call
- `self.failed_queue` already exists at line 134 — use it, do NOT create a new queue
- Use `item.get('attempts', 0) + 1` (not `item['attempts'] += 1`) in case existing items don't have the key
- `logger` is the module-level logger already in use in `resilience.py` — do not import a new one
- Do NOT add any socket emit or health check surface for dropped items — log only (per CONTEXT.md)

Read the actual method body before replacing to preserve any existing logic (timing, metrics, etc.) not mentioned above.
