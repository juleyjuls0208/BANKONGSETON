Endpoints at /cashier/* prefix:
- GET  /cashier/login               — login page

- GET  /cashier/api/products        — product grid (JWT-protected, uses active cashier session)
- POST /cashier/api/process-sale    — body { items, total }; initiates card reading
- POST /cashier/api/complete-sale   — body { card_uid }; finalizes transaction

## Transaction flow detail
process-sale stores pending transaction in Flask session
→ WebSocket event 'cashier_request_card' emitted
→ ArduinoBridge polls serial port (5-second timeout)
→ Arduino sends <CARD|ABCD1234> over serial at 9600 baud
→ complete-sale called with card_uid from WebSocket response
→ balance deducted from Money Accounts (col C)
→ transaction logged to Transactions Log (7-column format, no BalanceBefore)
→ response returned to cashier browser

## Arduino serial protocol
Expected format from Arduino: <CARD|ABCD1234>
Baud rate: 9600
Read timeout: 5 seconds (hardcoded in ArduinoBridge)

## Arduino wiring (from docs/context.md, verified against arduino_bridge.py serial expectations)
RC522 RFID reader on SPI:
- SS (SDA) → Arduino pin 10
- SCK → Arduino pin 13
- MOSI → Arduino pin 11
- MISO → Arduino pin 12
- RST → Arduino pin 9
- VCC → 3.3V CRITICAL (NOT 5V — will damage RC522)
- GND → GND

**H2: Hardware Setup**

**H3: Required Hardware**
- Arduino Uno (or compatible)
- RC522 RFID/NFC module
- USB cable (Arduino to PC)
- RFID cards/tags (ISO 14443A, 13.56 MHz)

**H3: Arduino Wiring**
Table: RC522 Pin | Arduino Pin | Notes
- SDA (SS) → Pin 10
- SCK → Pin 13
- MOSI → Pin 11
- MISO → Pin 12
- RST → Pin 9
- VCC → 3.3V (⚠️ NOT 5V — will damage the RC522 module)
- GND → GND

**H3: Arduino Firmware**
The Arduino must be flashed with firmware that reads RC522 cards and outputs them over serial in this exact format:
```
<CARD|ABCD1234>
```
Where `ABCD1234` is the 8-character hex card UID.
Serial settings: 9600 baud, 8N1.

**H2: Connecting to Arduino**
1. Plug Arduino into the computer running the dashboard server via USB
2. Open the cashier POS in browser
3. In the connection panel, click "Scan Ports" — this calls `GET /cashier/api/ports`
4. Select the correct COM port from the dropdown
5. Click "Connect" — this calls `POST /cashier/api/connect-arduino` with `{"port": "COM3"}`
6. Status indicator turns green when connected

**H2: Processing a Sale**

Step-by-step transaction flow:
1. After login, the product grid loads (`GET /cashier/api/products`)
2. Cashier clicks products to add them to the cart
3. Running total is displayed
4. Cashier clicks "Charge" — `POST /cashier/api/process-sale` with `{"items": [...], "total": 25.0}`
5. Server stores the pending transaction in Flask session and emits WebSocket event `cashier_request_card`
6. POS UI shows "Please tap card" prompt
7. Student taps RFID card on Arduino reader
8. Arduino sends `<CARD|ABCD1234>` over serial (9600 baud)
9. ArduinoBridge reads card UID (5-second timeout)
10. WebSocket response delivers card UID to browser
11. Browser calls `POST /cashier/api/complete-sale` with `{"card_uid": "ABCD1234"}`
12. Server validates UID (regex `^[0-9A-Fa-f]{8}$`), looks up student in Money Accounts, deducts balance, logs transaction
13. POS shows "Sale Complete" with new balance

**H2: Cashier API Endpoints**
Table or list of all `/cashier/*` endpoints (from reading cashier_routes.py):
- Method | Path | Description — for all 8 cashier endpoints

**H2: Product Categories**
Products are grouped in the POS grid by Category. Categories are defined when products are added in the admin dashboard. Standard categories: Food, Drinks, Snacks. Others can be defined freely.

**H2: Transaction Record**
Each completed sale is written to the Transactions Log Google Sheet with these fields (7 columns — note: no BalanceBefore in this write path):
Timestamp, MoneyCardNumber, TransactionType ('Purchase'), Amount (negative), BalanceAfter, Status ('Success'), ItemsJson

**H2: Troubleshooting**
- "Product grid is empty" → check Products sheet exists and has Active=TRUE rows; check cashier JWT token is valid (re-login)
- "Arduino not listed in ports" → check USB connection; on Windows, look in Device Manager for COM ports; on Linux, look for /dev/ttyUSB0
- "Card timeout: no card read after 5 seconds" → 5-second read timeout is hardcoded in ArduinoBridge; ensure Arduino is connected and outputting `<CARD|XXXXXXXX>` format
- "Card UID invalid error" → UID must be exactly 8 hex chars (ABCD1234); if Arduino firmware sends different format, it must be changed
- "Transaction failed: student not found" → card UID not in Money Accounts sheet
- "Transaction failed: insufficient balance" → student balance below transaction total

Cross-references: [API Reference](api-reference.md), [Google Sheets Schema](google-sheets-schema.md), [Admin Guide](admin-guide.md).
