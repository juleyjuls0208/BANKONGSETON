**H1: BankongSeton Architecture**

**H2: System Overview**
One-paragraph description: school canteen cashless payment system; students pay by tapping RFID card; backend is Python/Flask; database is Google Sheets; Android app shows balance and history.

**H2: High-Level Architecture Diagram**
ASCII diagram showing: Android App → API Server (5001) → Google Sheets; Admin Browser → Dashboard Server (5003) → Google Sheets; Cashier Browser → Dashboard Server (5003) → Arduino Bridge → RFID Reader.

**H2: Server Architecture**
Two servers — explain each:
- **API Server** (`backend/api/api_server.py`, port 5001): Student-facing REST API. JWT + session token auth. Handles: login, balance, transactions, FCM token, products, NFC, cashier transaction endpoint.
- **Dashboard Server** (`backend/dashboard/admin_dashboard.py`, port 5003): Admin web dashboard + cashier POS (Blueprint). Session-based web auth. Manages products, students, load balance, transaction monitoring. Cashier POS is a sub-Blueprint at `/cashier/*`.

Both must run simultaneously. Mobile app connects to port 5001. Browser-based tools connect to port 5003.

**H2: Layers**
Describe the 5 layers:
1. Client Layer: Android app + Admin browser + Cashier browser
2. Transport Layer: HTTP/REST (mobile) + HTTP/WebSocket (cashier POS)
3. Application Layer: Flask API server + Flask Dashboard server
4. Persistence Layer: Google Sheets via gspread (google-auth service account)
5. Hardware Layer: Arduino RFID reader via serial port (COM port)

**H2: Authentication**
Two parallel auth mechanisms — distinguish them clearly:
- **Session Token** (student endpoints): `POST /api/auth/login` returns a token stored in `active_sessions` dict. Subsequent requests: `Authorization: Bearer <token>`. Validated as `token not in active_sessions` (dict lookup, NOT JWT decode).
- **JWT** (cashier/admin API endpoints): `@require_auth(roles=['admin', 'cashier'])` decorator. Token is a signed JWT (HS256). Used by `/api/cashier/transaction`, `/api/products` (POST), `/api/nfc/pay`.
- **Web Session** (admin dashboard): Flask session cookies for browser-based admin login.

**H2: Data Flow — Card Payment**
Step-by-step: Student taps card → Arduino reads UID → serial `<CARD|ABCD1234>` → ArduinoBridge → cashier_routes.py validates UID (regex `^[0-9A-Fa-f]{8}$`) → looks up Money Accounts sheet → deducts balance → writes Transactions Log → returns response.

**H2: Data Flow — Mobile Balance Check**
Student opens app → `GET /api/student/balance` with session Bearer token → api_server.py validates token → reads Money Accounts sheet → returns `{ "balance": 250.00 }`.

**H2: Google Sheets as Database**
Explain why (school project, no SQL migration needed): 7 sheets: Users, Money Accounts, Transactions Log, Lost Card Reports, Products, VirtualCards, Settings. Cross-reference to `docs/google-sheets-schema.md` for column details.

**H2: Key Entry Points**
Table: File | Role | Port | Start Command
- `backend/api/api_server.py` | Student API | 5001 | `python backend/api/api_server.py`
- `backend/dashboard/admin_dashboard.py` | Admin Dashboard + Cashier POS | 5003 | `python backend/dashboard/admin_dashboard.py`

**H2: Key Source Files**

**H1: Setup Guide**

**H2: Prerequisites**

**H2: 1. Clone the Repository**
```bash
git clone <repo-url>
cd BANKONGSETON
```

**H2: 2. Install Python Dependencies**
Two sets of dependencies. Both must be installed:
```bash
# API server dependencies
pip install -r backend/api/requirements_api.txt

# Dashboard dependencies
pip install -r backend/dashboard/requirements.txt
```
Note: Use `google-auth` (already in requirements). Do NOT install `oauth2client` — it's deprecated and was removed in Phase 2.

**H2: 3. Set Up Google Sheets**
Step-by-step with exact sub-steps:
1. Create a new Google Spreadsheet at sheets.google.com. Copy the spreadsheet ID from the URL (`https://docs.google.com/spreadsheets/d/<SPREADSHEET_ID>/edit`).
2. Enable Google Sheets API:
   - Go to console.cloud.google.com
   - Create or select a project
   - Enable "Google Sheets API"
3. Create a service account:
   - APIs & Services → Credentials → Create Credentials → Service Account
   - Grant role: Editor
   - Create key: JSON format → download
4. Rename the downloaded JSON to `credentials.json` and place it at `config/credentials.json`.
5. Share the Google Spreadsheet with the service account email (found in credentials.json as `client_email`). Give it Editor access.
6. Create the following sheets (tabs) in the spreadsheet with exact header rows:

| Sheet Name | Columns (in order) |
|---|---|
| Users | StudentID, Name, IDCardNumber, MoneyCardNumber, Status, ParentEmail, DateRegistered, FCMToken |
| Money Accounts | MoneyCardNumber, StudentIDCard, Balance, Status, LastUpdated |
| Transactions Log | Timestamp, MoneyCardNumber, TransactionType, Amount, BalanceBefore, BalanceAfter, Status, ItemsJson |
| Lost Card Reports | (see docs/google-sheets-schema.md) |
| Settings | Key, Value |

Note: Products and VirtualCards sheets are created automatically on first use.
Note: FCMToken is the 8th column in Users — it was added in Phase 4. Must be present for push notifications.

**H2: 4. Configure Environment Variables**
Copy `.env.example` to `.env`:
```bash
cp .env.example .env
```
Edit `.env` and fill in:
```bash
# REQUIRED — do not leave blank or the server will refuse to start
FLASK_SECRET_KEY=your-random-secret-key-here
GOOGLE_SHEETS_ID=your-spreadsheet-id-from-step-3
GOOGLE_CREDENTIALS_FILE=config/credentials.json

# Server ports
API_PORT=5001
DASHBOARD_PORT=5003

# Admin credentials
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your-admin-password

# CORS — comma-separated allowed origins
# For development: leave empty to allow all localhost origins
CORS_ORIGINS=

# Optional: Flask environment
FLASK_ENV=development
```
List all 17 env vars from `.env.example` with a brief description of each. Group them: Server Config, Google Sheets, Auth, CORS, Optional.

**H2: 5. Start the Servers**
IMPORTANT: Run both servers. They are separate processes on separate ports. Open two terminal windows:

Terminal 1 — API Server (for Android app):

python backend/api/api_server.py

Access points:

**H2: 6. Android App Setup**
1. Open `mobile/student_app_v2/` in Android Studio.
2. Edit `ApiClient.kt` line 38: change `BASE_URL` to your API server's IP address and port:
   ```kotlin
   private const val BASE_URL = "http://192.168.1.100:5001/"
   ```
   Use your machine's local IP (not `localhost` — the emulator/phone needs a real IP).
3. Firebase (FCM push notifications): Replace `mobile/student_app_v2/google-services.json` with your own from your Firebase project console. This file in the repo is a placeholder for the original dev environment.
4. Build and run on an Android device or emulator.

**H2: 7. Arduino Setup**
Brief: Flash Arduino with the RFID sketch (not included in this repo — see docs/cashier-guide.md for wiring). Connect via USB. In the cashier POS browser, select the COM port and click Connect.

**H2: Verify Everything Is Working**
Quick smoke test checklist:
1. `curl http://localhost:5001/api/health` returns `{"status": "healthy"}`
2. Open http://localhost:5003/admin/ — admin login page appears
3. Open http://localhost:5003/cashier/login — cashier login page appears
4. Admin login with credentials from ADMIN_USERNAME/ADMIN_PASSWORD works

**H2: Troubleshooting**
Cover at minimum:
- "Server refuses to start: FLASK_SECRET_KEY" → must be non-empty in .env
- "Google Sheets API error: credentials" → check file path in GOOGLE_CREDENTIALS_FILE, check service account has Editor access on the sheet
- "401 Unauthorized on /api/auth/login" → check student ID exists in Users sheet, check MoneyCardNumber in Money Accounts sheet
- "Android app cannot connect" → BASE_URL in ApiClient.kt must be machine's local IP, not localhost; both servers must be running
- "Products sheet not appearing in admin" → Products sheet is auto-created on first POST /api/products or admin dashboard product page visit
- "pip install fails on google-auth" → requires Python 3.8+; do NOT install oauth2client (removed)
- "CORS error in browser" → add your browser origin to CORS_ORIGINS in .env
