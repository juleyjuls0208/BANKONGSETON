# Phase 20.1 Plan 01: Arduino PN532 NFC Firmware (APDU + R4 Swap) Summary

**One-liner:** Dual-mode APDU/UID-fallback firmware for R3 and PN532 swap on R4 posting JSON to `/api/nfc/pay`

## What Was Built

### Task 1 — R3 APDU Dual-Mode Firmware (`da9ed10`)
Modified `arduino/bankongseton_nfc_r3/bankongseton_nfc_r3.ino` to attempt a SELECT AID APDU exchange after detecting a passive NFC target:

- Idle LCD changed `"Tap card..."` → `"Tap Phone..."` to cue HCE phone taps
- After `readPassiveTargetID()` succeeds, LCD shows `"Reading..."` before APDU
- `uint8_t apduCmd[19]` with BANKONGSETON AID bytes declared inline
- `nfc.inDataExchange(apduCmd, 19, response, &responseLength)` called on the already-listed target
- APDU success path: validates `responseLength==50 && response[48]==0x90 && response[49]==0x00`, emits `NFC|<48-char-token>`, LCD shows `"OK"` / `"Payment sent"`
- APDU failure path: emits `CARD|<uidHex>` (existing fallback), LCD shows `"NFC Error"` / `"Using card UID"`

### Task 2 — R4 PN532 Swap + `/api/nfc/pay` Retarget (`23b0fc7`)
Replaced MFRC522 library with Adafruit PN532 in `arduino/bankongseton_rfid/bankongseton_rfid.ino`:

- `#include <MFRC522.h>`, `RST_PIN 9`, `MFRC522 mfrc522(SS_PIN, RST_PIN)` → `#include <Adafruit_PN532.h>`, `PN532_SS 10`, `Adafruit_PN532 nfc(PN532_SS)`
- `setup()`: `mfrc522.PCD_Init()` → `nfc.begin(); nfc.SAMConfig()`
- `loop()`: MFRC522 scan block → PN532 `readPassiveTargetID()` + inline hex builder → `deliverUID(uidStr)`
- `httpPost()`: path `/api/arduino/card-read` → `/api/nfc/pay`, body key `"uid"` → `"token"`, Content-Type set to `application/json`
- `deliverUID()`, `ensureWiFi()`, `connectWiFi()`, `MAX_RETRIES`, retry logic all **unchanged**

### Task 3 — Checkpoint Human Verify
⚡ Auto-approved (yolo mode) — code review confirms both sketches structurally correct. Hardware upload recommended before deployment.
