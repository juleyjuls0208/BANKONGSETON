  <action>
    Read SettingsActivity.kt to understand existing structure: how it retrieves the auth token (likely from SecureStorage or SharedPreferences), existing button click patterns, and lifecycle methods.

    Add the following logic:

    **In onCreate (after existing view bindings):**
    ```kotlin
    val nfcSection = findViewById<LinearLayout>(R.id.nfcSection)   // or ViewGroup
    val nfcStatusText = findViewById<TextView>(R.id.nfcStatusText)
    val setupNfcButton = findViewById<MaterialButton>(R.id.setupNfcButton)
    val removeNfcButton = findViewById<MaterialButton>(R.id.removeNfcButton)

    // Show section only on NFC-capable devices
    if (NfcManager.isNfcAvailable(this)) {
        nfcSection.visibility = View.VISIBLE
    }
    // else: stays gone — non-NFC devices see nothing

    fun refreshNfcSection() {
        if (NfcManager.isRegistered(this)) {
            nfcStatusText.text = "✓ NFC Payment Ready"
            setupNfcButton.visibility = View.GONE
            removeNfcButton.visibility = View.VISIBLE
        } else {
            nfcStatusText.text = "Not set up"
            setupNfcButton.visibility = View.VISIBLE
            removeNfcButton.visibility = View.GONE
        }
    }
    refreshNfcSection()

    setupNfcButton.setOnClickListener {
        val authToken = /* retrieve from SecureStorage same way other activities do */ ""
        NfcManager.registerDevice(this, authToken) { success ->
            runOnUiThread {
                if (success) {
                    refreshNfcSection()
                } else {
                    Toast.makeText(this, "Registration failed. Please try again.", Toast.LENGTH_SHORT).show()
                }
            }
        }
    }

    removeNfcButton.setOnClickListener {
        val authToken = /* retrieve from SecureStorage */ ""
        NfcManager.unregisterDevice(this, authToken) { success ->
            runOnUiThread {
                if (success) {
                    refreshNfcSection()
                } else {
                    Toast.makeText(this, "Could not remove NFC device. Please try again.", Toast.LENGTH_SHORT).show()
                }
            }
        }
    }
    ```

    **Auth token retrieval:** Use the exact same pattern already used in SettingsActivity or HomeActivity for auth-protected API calls (read those files to find the pattern — typically `SecureStorage.getAuthToken(this)` or from SharedPreferences).

    **NfcManager method signatures:** NfcManager.registerDevice takes `(context, pin, authToken, callback)` — the biometric prompt is launched internally by NfcManager, and PIN is collected via its internal `showPinDialog`. So from SettingsActivity you only pass `context` and `authToken` (NfcManager handles the PIN prompt and biometric). Adjust the call to match the actual NfcManager signature from Plan 01.

    Also override `onResume` to call `refreshNfcSection()` so the state updates if user navigates away and back.
