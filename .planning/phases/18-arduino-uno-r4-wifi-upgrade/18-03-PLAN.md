  <action>
Run the Flask server (if not already running) and execute these curl checks to verify the endpoint behaves correctly. All three checks must produce the expected HTTP status codes.

**Start Flask (if needed):**
```bash
cd backend/dashboard
python web_app.py
```

**Check 1 — Missing API key returns 401:**
```bash
curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:5000/api/arduino/card-read \
  -H "Content-Type: application/json" \
  -d '{"uid":"ABCDEF12"}'
# Expected: 401
```

**Check 2 — Wrong API key returns 401:**
```bash
curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:5000/api/arduino/card-read \
  -H "Content-Type: application/json" \
  -H "X-API-Key: wrong-key" \
  -d '{"uid":"ABCDEF12"}'
# Expected: 401
```

**Check 3 — Invalid UID returns 400:**
```bash
curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:5000/api/arduino/card-read \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $ARDUINO_API_KEY" \
  -d '{"uid":"TOOSHORT"}'
# Expected: 400
```

**Check 4 — Valid request returns 200 (requires ARDUINO_API_KEY set in env):**
```bash
curl -s -X POST http://localhost:5000/api/arduino/card-read \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $ARDUINO_API_KEY" \
  -d '{"uid":"ABCDEF12"}'
# Expected: {"status": "ok"} with HTTP 200
```

Note: For Check 4, set a non-empty `ARDUINO_API_KEY` in `.env` (copy `.env.example` and fill it in). If Flask is running without `ARDUINO_API_KEY` set, Checks 1-3 will still pass (endpoint correctly disabled), but Check 4 will return 401 — which is correct behavior for unset key.
