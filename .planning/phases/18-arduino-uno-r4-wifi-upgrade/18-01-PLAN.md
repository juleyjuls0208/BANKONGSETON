In `web_app.py`, add the following **at module level** (near line 204, after the `UID_PATTERN` definition and `validate_card_uid()` function):

```python
# Arduino WiFi API key — loaded once at module level
# Empty string means endpoint is disabled (returns 401 for any request)
ARDUINO_API_KEY = os.environ.get('ARDUINO_API_KEY', '')
```

Then add the route handler. Place it in the **Arduino/Serial section** of the file (search for the `@app.route` handlers near the existing serial port routes — around the `connect_arduino`, `disconnect_arduino`, `get_ports` handlers). Insert after the last Arduino-related route:

```python
@app.route('/api/arduino/card-read', methods=['POST'])
def arduino_card_read():
    """
    WiFi card-read endpoint for Arduino UNO R4 WiFi.
    Arduino POSTs {"uid": "ABCDEF12"} with X-API-Key header.
    Emits card_read SocketIO event — identical to the serial path in ArduinoBridge.
    ARDW-02
    """
    # 1. Validate API key — reject empty ARDUINO_API_KEY to prevent open-door bug
    api_key = request.headers.get('X-API-Key', '')
    if not ARDUINO_API_KEY or api_key != ARDUINO_API_KEY:
        logger.warning("event=arduino_card_read_rejected reason=invalid_api_key remote=%s",
                       request.remote_addr)
        return jsonify({'error': 'Unauthorized'}), 401

    # 2. Validate UID format (same UID_PATTERN used throughout web_app.py)
    data = request.get_json(silent=True) or {}
    uid = data.get('uid', '')
    if not UID_PATTERN.match(uid):
        logger.warning("event=arduino_card_read_rejected reason=invalid_uid uid=%r", uid)
        return jsonify({'error': 'Invalid UID format — expected 8 hex chars'}), 400

    # 3. Emit card_read event — same shape as ArduinoBridge._read_card_thread
    #    Cashier frontend receives this identically to a serial card read (ARDW-03)
    socketio.emit('card_read', {'success': True, 'uid': uid})
    logger.info("event=arduino_card_read uid=%s remote=%s", uid, request.remote_addr)

    return jsonify({'status': 'ok'}), 200
```

**Critical implementation notes:**
- `ARDUINO_API_KEY` is read at module import time via `os.environ.get`. `load_dotenv()` is called before `web_app` is imported (confirmed in `wsgi.py` pattern from Phase 07.1), so this is safe.
- The empty-ARDUINO_API_KEY guard (`if not ARDUINO_API_KEY`) prevents the pitfall where an unset env var causes `'' == ''` to match any request (documented in RESEARCH.md Pitfall 4).
- The SocketIO emission shape `{'success': True, 'uid': uid}` matches exactly what `ArduinoBridge._read_card_thread` emits (arduino_bridge.py lines 75-78).
- No `normalize_card_uid()` call needed here — the Arduino firmware sends uppercase hex (`%02X`), and `UID_PATTERN` accepts both cases. The endpoint is additive; normalisation happens downstream in existing transaction handlers.
- Do NOT emit `card_read` conditionally based on `card_reader_state` — the existing serial path doesn't guard on state either; the cashier JS handles unexpected events gracefully.
