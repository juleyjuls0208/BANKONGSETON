Create backend/utils.py with the following content:

1. **normalize_card_uid(uid) -> str**
   - Merges both existing implementations (admin_dashboard.py line 196 and api_server.py line 144)
   - Canonical logic: if not uid → return ""; else return str(uid).strip().lstrip('0').upper()
   - The admin version handles None (returns ""); the api version does not — the canonical version MUST handle None to avoid crashes
   - Do NOT import from errors.py here (circular import risk); use logging.getLogger('bangko.utils') directly

2. **CardReaderState class** with a single threading.Lock:
   - __init__: self._lock = threading.Lock(); self.arduino = None; self.arduino_bridge = None; self.card_reading_active = False; self.pending_student_id = None
   - get(self, key: str) -> any: with self._lock: return getattr(self, key)
   - set(self, key: str, value) -> None: with self._lock: setattr(self, key, value)
   - update(self, **kwargs) -> None: with self._lock: for k,v in kwargs.items(): setattr(self, k, v)
   - All four globals covered: arduino, arduino_bridge, card_reading_active, pending_student_id

3. **Module-level singleton**: card_reader_state = CardReaderState()

4. **Module docstring** at top explaining both exports.

5. **No imports from errors.py** — use logging.getLogger('bangko.utils') directly for any internal logging in utils.py to avoid circular imports.

AVOID: importing from admin_dashboard.py or errors.py — this file must be importable standalone.
USE: threading.Lock (not RLock — re-entrant locking not needed here since get/set don't call each other).
