<objective>
Give cashiers visibility into their current shift's performance: how much money collected, how many transactions processed, and how many items sold.

Stats are tracked in the Flask server-side session (no new Google Sheet required). They reset on login and can be manually reset via a button. The panel updates automatically after each completed sale.

Purpose: Cashiers currently have no way to verify their shift total without manually counting.
Output: Shift counter logic in `cashier_routes.py` + summary panel in `cashier_index.html`.
</objective>

```python
@cashier_bp.route('/api/shift/summary', methods=['GET'])
def get_shift_summary():
    if not session.get('cashier_logged_in'):
        return jsonify({'success': False, 'error': 'Not authenticated'}), 401
    return jsonify({
        'success': True,
        'total_sales': session.get('shift_total_sales', 0.0),
        'transaction_count': session.get('shift_transaction_count', 0),
        'items_sold': session.get('shift_items_sold', 0)
    })


@cashier_bp.route('/api/shift/reset', methods=['POST'])
def reset_shift():
    if not session.get('cashier_logged_in'):
        return jsonify({'success': False, 'error': 'Not authenticated'}), 401
    session['shift_total_sales'] = 0.0
    session['shift_transaction_count'] = 0
    session['shift_items_sold'] = 0
    session.modified = True
    return jsonify({'success': True, 'message': 'Shift counters reset'})
```

```javascript
async function refreshShiftSummary() {

    const res = await fetch('/cashier/api/shift/summary');
```
