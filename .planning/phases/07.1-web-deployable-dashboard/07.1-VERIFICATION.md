**Why human:** Requires live Google Sheets connection for `GET /cashier/api/lookup-student` to return real student records.

**Test:** Select a student from the lookup results and confirm payment.
**Expected:** Transaction appears in "Transactions Log" sheet with: column 1 = TXN-xxxxxx, column 5 = `Manual`, column 6 = positive amount, column 9 = `Completed`, column 10 = empty string.
**Why human:** Requires live Sheets write and visual inspection of the spreadsheet.

#### 4. Web Mode Badge — Visual Appearance

**Test:** Load `/dashboard` in a browser when `arduino_available=False` (i.e., when running under `web_app.py`).
**Expected:** Subtle semi-transparent pill badge reading "Web Mode" appears next to "Banko ng Seton" in the sidebar brand area, styled in small (11px) white text.
**Why human:** Requires browser rendering of the Jinja2 template to confirm visual output.

#### 5. RFID Hardware Mode Unchanged

**Expected:** Port selector shows available COM ports; connect button works; card tap triggers `checkout()` → `completeSale(cardUid)` flow as before; SocketIO is initiated normally.
