**Step 1 — BankoHceService.kt: Make fields private, add accessor methods:**

Find these two field declarations:
```kotlin
@Volatile var isPaymentAuthorized: Boolean = false
@Volatile var currentToken: String? = null
```

Change them to `private`:
```kotlin
@Volatile private var isPaymentAuthorized: Boolean = false
@Volatile private var currentToken: String? = null
```

IMPORTANT: Keep `@Volatile` on both fields — this is required for thread safety.

After the field declarations (or as companion methods — wherever is natural in this class), add these three public methods:

```kotlin
fun authorize(token: String) {
    currentToken = token
    isPaymentAuthorized = true
}

fun deauthorize() {
    isPaymentAuthorized = false
    currentToken = null
}

fun reauthorize() {
    // Only re-authorize if a token is already loaded (safe guard)
    if (currentToken != null) {
        isPaymentAuthorized = true
    }
}

fun isAuthorized(): Boolean = isPaymentAuthorized
```

If `BankoHceService` is a singleton or companion object, add these as instance methods (not static). Check how `NfcManager` and `NfcPayOverlayActivity` currently reference the service instance and use the same access pattern.

**Step 2 — NfcManager.kt: Replace all direct field writes with method calls:**
