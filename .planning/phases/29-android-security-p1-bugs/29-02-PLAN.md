  <name>Task 1: Reset isClickable at top of bind() to fix ViewHolder reuse bug (REQ-BUG-MOB-07)</name>

  <action>
In `TransactionsAdapter.kt`, add `itemView.isClickable = true` as the **first line** of `bind()`, before any other logic.

The root cause: when a non-purchase row runs `itemView.isClickable = false`, RecyclerView recycles that ViewHolder. If it is later reused for a Purchase row, `isClickable` is still `false` — the `setOnClickListener` is attached but the touch event is never dispatched. The fix is to unconditionally reset `isClickable` to `true` at the start of every `bind()` call, before the `isPurchase` branch.

Find this line:
```kotlin
        fun bind(transaction: Transaction) {
            typeText.text = transaction.type
```

Change to:
```kotlin
        fun bind(transaction: Transaction) {
            itemView.isClickable = true
            typeText.text = transaction.type
```

Do NOT remove the `itemView.setOnClickListener(null)` and `itemView.isClickable = false` lines in the else-branch — they are still needed for the current render of a non-purchase row. Only the recycling path is broken; the reset at the top of bind() fixes it.
  </action>
