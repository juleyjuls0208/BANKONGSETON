## Summary

Phase 32.1 goal is **achieved** at the code level. All four requirements (NFC-FIX-01 through NFC-FIX-04) are fully implemented:

1. **R4 firmware** (`bankongseton_rfid.ino`): Both `setRFField(1,1)` bug-calls replaced with `setRFField(0,1)`. Retry limiter removed. NTAG215 can now power up in passive sensing mode.

2. **R3 firmware** (`bankongseton_nfc_r3.ino`): RF toggle backported. 7-byte UIDs (NTAG215) correctly routed to the `CARD|<uid>` serial path rather than the APDU path. Diagnostic hex dump added for no-card events.

3. **ArduinoBridge** (`arduino_bridge.py`): Fully rewritten. Parses `NFC|`, `CARD|`, `ERROR|` wire formats. All 8 methods required by `dashboard_core.py` and the test suite are present. Old `<CARD|>` angle-bracket format removed.

4. **Cashier validation** (`cashier_routes.py`): `UID_PATTERN` widened from 8-char-only to accept 8-char or 14-char hex UIDs. NTAG215 payments will no longer be rejected with HTTP 400.

Two non-blocking documentation gaps noted above (REQUIREMENTS.md entries missing; ROADMAP checkboxes not updated). Hardware verification requires a physical card and running hardware — cannot be automated.
