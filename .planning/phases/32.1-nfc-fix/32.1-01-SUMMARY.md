# Phase 32.1: NFC Fix Summary

**Fixed PN532 RF field configuration in R4 sketch: autoRFCA 1→0 at both RF-on sites + removed 10-retry limit, enabling NTAG215 passive ISO14443A card detection**

## Accomplishments
- Replaced both `setRFField(1, 1)` (autoRFCA=1, active/P2P mode) with `setRFField(0, 1)` (autoRFCA=0, passive mode) so PN532 does not suppress its own RF field during passive sensing
- Removed `setPassiveActivationRetries(0x0A)` from setup() — 10 retries are insufficient for NTAG215's 7-byte double-cascade anticollision; PN532 default 0xFF bounded by NFC_TIMEOUT_MS is correct
- All `setRFField(0, 0)` RF-off calls preserved unchanged; `NFC_TIMEOUT_MS` constant and usage preserved

## Decisions Made
- autoRFCA=0 required for passive ISO14443A — autoRFCA=1 is for NFC peer-to-peer active mode; using it suppresses the PN532's own RF field so NTAG215 never powers up, yielding NbTg=0
- Remove retry limit rather than increase it — PN532 default 0xFF with NFC_TIMEOUT_MS timeout is the correct pattern; the comment block explaining the old (incorrect) reasoning was also removed
