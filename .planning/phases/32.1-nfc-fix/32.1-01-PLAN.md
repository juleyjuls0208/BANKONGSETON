# Phase 32.1: nfc-fix — Plan 01

**Phase:** 32.1 — NFC Fix (INSERTED URGENT)
**Plan:** 01 of 01
**Goal:** Fix NTAG215 physical card detection (PN532 firmware) and align ArduinoBridge to support both R3/R4 readers with correct wire format.
**Estimated tasks:** 7
**Dependencies:** None — all changes are surgical edits to existing files.

---

## Context

Physical NTAG215 student NFC cards are not detected at the canteen. Two bugs block end-to-end operation:

1. **Arduino firmware:** `setRFField(1, 1)` (autoRFCA=1) suppresses RF during passive ISO14443A sensing → NTAG215 never powers up → NbTg=0. R4 has the bug in two places; R3 is missing the RF toggle entirely.
2. **ArduinoBridge:** Parses obsolete `<CARD|...>` angle-bracket format instead of `CARD|<uid>` and `NFC|<token>`. Single serial port; both R3 and R4 are deployed at the canteen.
3. **Cashier route:** `UID_PATTERN` only accepts 8-char UIDs; NTAG215 produces 14-char UIDs → NTAG215 payments rejected at the cashier payment endpoint even after firmware fix.

All root causes are confirmed by direct code inspection. No new libraries required.


### WAVE 2 — ArduinoBridge Rewrite

---

#### Task 04 — Rewrite arduino_bridge.py

**File:** `backend/dashboard/arduino_bridge.py`

**What:** Full rewrite of the class. Preserve the existing public API
(`read_card_with_timeout`, `cancel_reading`). Add `_parse_line`, `_start_serial_thread`,
`start_background_listener`, `_post_nfc_payment`, `_timeout_watcher`.

**Why:**
- Current code parses obsolete `<CARD|ABCD1234>` angle-bracket format; sketches emit `CARD|<uid>` and `NFC|<token>`
- `start_background_listener()` is called by `dashboard_core.py` line 357 but doesn't exist → AttributeError
- Test suite (`test_arduino_bridge_nfc.py`) expects `_parse_line`, `_start_serial_thread`, `_post_nfc_payment` as named methods
- Tests patch `_start_serial_thread` in constructor — it **must** be a named method called from `__init__`

**New implementation:**

```python
"""
Arduino NFC Bridge — multi-format serial reader for BANKONGSETON canteen system.

Supports wire formats emitted by both R3 and R4 sketches:
  NFC|<48-char-token>   — HCE phone payment token
  CARD|<uid-hex>        — physical NFC card UID (NTAG215 or 4-byte fallback)
  ERROR|<reason>        — sketch-side error

Starts a background read thread in __init__ via _start_serial_thread().
Supports read_card_with_timeout() for cashier POS card-tap flow.
"""

import os
import time
import threading
import requests


class ArduinoBridge:
    def __init__(self, arduino_serial, socketio):
        self.arduino = arduino_serial
        self.socketio = socketio
        self.reading_active = False
        self.current_callback = None
        self.timeout_seconds = 5
        self._stop_thread = False
        self._thread = None
        self._start_serial_thread()

    # ── Background serial reader ─────────────────────────────────

    def _start_serial_thread(self):
        """Start background thread that continuously reads from the serial port.
        Called from __init__; also patchable in tests via patch.object."""
        self._stop_thread = False
        self._thread = threading.Thread(target=self._serial_loop, daemon=True)
        self._thread.start()

    def start_background_listener(self):
        """Called by dashboard_core.py after construction.
        Thread is already running from __init__; this is a no-op unless the
        thread died (e.g. serial disconnected and reconnected)."""
        if self._thread is None or not self._thread.is_alive():
            self._start_serial_thread()

    def _serial_loop(self):
        """Loop: read lines from serial port, dispatch to _parse_line."""
        while not self._stop_thread:
            if not self.arduino or not self.arduino.is_open:
                time.sleep(0.5)
                continue
            try:
                if self.arduino.in_waiting > 0:
                    raw = self.arduino.readline()
                    line = raw.decode('utf-8', errors='ignore').strip()
                    if line:
                        self._parse_line(line)
                else:
                    time.sleep(0.05)
            except Exception as e:
                print(f"[ArduinoBridge] serial read error: {e}")
                time.sleep(0.5)

    # ── Line parser ──────────────────────────────────────────────

    def _parse_line(self, line: str):
        """Parse one serial line and emit appropriate SocketIO events.

        Wire formats (both R3 and R4):
          NFC|<48-char-token>   → emits "nfc_payment" + POSTs to /api/nfc/pay
          CARD|<uid-hex>        → emits "card_read"; unblocks read_card_with_timeout
          ERROR|<reason>        → emits "nfc_payment_result" with success=False
        """
        if line.startswith("NFC|"):
            token = line[4:]
            self.socketio.emit("nfc_payment", {"token": token})
            self._post_nfc_payment(token)

        elif line.startswith("CARD|"):
            uid = line[5:]
            self.socketio.emit("card_read", {"success": True, "uid": uid})
            if self.reading_active and self.current_callback:
                self.reading_active = False
                cb = self.current_callback
                self.current_callback = None
                cb(uid)

        elif line.startswith("ERROR|"):
            error = line[6:]
            self.socketio.emit("nfc_payment_result", {"success": False, "error": error})

    # ── NFC payment HTTP delivery ────────────────────────────────

    def _post_nfc_payment(self, token: str):
        """POST NFC token to /api/nfc/pay. Requires CASHIER_JWT env var.
        Silently skips if CASHIER_JWT is not set (dev/test environments)."""
        jwt_token = os.environ.get("CASHIER_JWT", "")
        base_url = os.environ.get("API_BASE_URL", "http://127.0.0.1:5001")
        if not jwt_token:
            print("[ArduinoBridge] CASHIER_JWT not set — skipping HTTP delivery")
            return
        try:
            requests.post(
                f"{base_url}/api/nfc/pay",
                json={"token": token},
                headers={"Authorization": f"Bearer {jwt_token}"},
                timeout=5,
            )
        except Exception as e:
            print(f"[ArduinoBridge] _post_nfc_payment error: {e}")

    # ── Cashier POS card-tap flow ────────────────────────────────

    def read_card_with_timeout(self, callback, timeout=5):
        """Arm the bridge to deliver the next CARD| read to `callback`.

        The background thread is already running; this just sets the callback
        and starts a timeout watcher. If no card arrives within `timeout`
        seconds, emits 'card_timeout'.

        Used by dashboard_core.py SocketIO handler.
        """
        if not self.arduino or not self.arduino.is_open:
            self.socketio.emit('card_error', {'message': 'Arduino not connected'})
            return False

        self.reading_active = True
        self.current_callback = callback
        self.timeout_seconds = timeout

        watcher = threading.Thread(target=self._timeout_watcher, daemon=True)
        watcher.start()
        return True

    def _timeout_watcher(self):
        """Emit card_timeout if reading_active is still True after timeout_seconds."""
        time.sleep(self.timeout_seconds)
        if self.reading_active:
            self.reading_active = False
            self.current_callback = None
            self.socketio.emit('card_timeout', {
                'message': f'No card detected within {self.timeout_seconds} seconds'
            })

    def cancel_reading(self):
        """Cancel active card-tap wait (called by cashier POS on dialog close)."""
        self.reading_active = False
        self.current_callback = None
```

**Verification:**
- [ ] `_start_serial_thread` is called from `__init__` (not inlined)
- [ ] `start_background_listener()` exists and is a no-op when thread is alive
- [ ] `_parse_line("NFC|" + "A"*48)` emits `"nfc_payment"` event
- [ ] `_parse_line("CARD|ABCD1234")` emits `"card_read"` and calls `current_callback`
- [ ] `_parse_line("ERROR|NFC_FAIL")` emits `"nfc_payment_result"` with `success=False`
- [ ] `read_card_with_timeout` returns `False` when serial not connected
- [ ] `read_card_with_timeout` returns `True` and starts watcher thread when connected
- [ ] `cancel_reading()` clears `reading_active` and `current_callback`
- [ ] Old `<CARD|...>` angle-bracket format is completely removed
- [ ] `from flask_socketio import emit` import is removed (was unused — we call `self.socketio.emit`)

#### Task 05 — Fix cashier_routes.py UID_PATTERN to accept 14-char UIDs

**File:** `backend/dashboard/cashier/cashier_routes.py`

**What:** Change `UID_PATTERN` on line 26 to also accept 14-character hex strings.

**Current:**
```python
UID_PATTERN = re.compile(r'^[0-9A-Fa-f]{8}$')
```

**Replace with:**
```python
UID_PATTERN = re.compile(r'^[0-9A-Fa-f]{8}$|^[0-9A-Fa-f]{14}$')
```

**Why:** NTAG215 physical cards have 7-byte UIDs, which produce 14-character hex strings.
The current pattern only accepts 8-character UIDs (4-byte cards / HCE phones). Without this fix,
NTAG215 UIDs will be rejected with a 400 error at the cashier payment endpoint even after
the firmware fix makes detection work. `dashboard_core.py` already has the correct pattern.
