In `cashier_routes.py`, the `complete_sale()` function currently does:
1. `money_sheet.update_cell(account_row, 3, new_balance)` (line 226 -- deducts balance)
2. `trans_sheet.append_row(transaction_row)` (line 242 -- logs transaction)

If step 2 fails, the balance is deducted but no transaction is recorded. Per user's locked decision:
- Strategy: retry-then-abort (3 attempts total, 2 retries)
- If all retries fail: cashier sees "Transaction failed -- please try again"
- Log failure only after all retries exhausted
- No partial state: either both succeed or neither does

**Replace the balance deduction + transaction log section (approximately lines 224-274)** with:

```python
# Transaction with retry and rollback (BUG-05)
import time
MAX_RETRIES = 3
last_exception = None

for attempt in range(MAX_RETRIES):
    try:
        # Step 1: Deduct balance
        money_sheet.update_cell(account_row, 3, new_balance)

        # Step 2: Log transaction
        trans_sheet = db.worksheet('Transactions Log')
        timestamp = get_philippines_time().strftime('%Y-%m-%d %H:%M:%S')

        transaction_row = [
            timestamp,
            normalized_card,
            'Purchase',
            -total,
            new_balance,
            'Success',
            json.dumps(items)
        ]

        trans_sheet.append_row(transaction_row)

        # Both succeeded -- clear pending and return success
        flask_session.pop('pending_transaction', None)

        # Send email (non-fatal, outside retry loop)
        try:
            users_sheet = db.worksheet('Users')
            user_records = users_sheet.get_all_records()

            for user in user_records:
                if normalize_card_uid(user.get('MoneyCardNumber', '')) == normalized_card:
                    sys.path.insert(0, os.path.join(os.path.dirname(__file__), '..', '..', 'services'))
                    from email_service import EmailService

                    email_service = EmailService()
                    email_service.send_receipt(
                        user.get('ParentEmail', ''),
                        user.get('Email', ''),
                        user.get('Name', 'Student'),
                        items,
                        total,
                        new_balance
                    )
                    break
        except Exception as email_err:
            logger.warning(f"Email send error (non-fatal): {email_err}")

        return jsonify({
            'success': True,
            'new_balance': new_balance,
            'timestamp': timestamp
        })

    except Exception as e:
        last_exception = e

        # Attempt rollback: restore original balance
        try:
            money_sheet.update_cell(account_row, 3, current_balance)
        except Exception as rollback_err:
            logger.error(f"CRITICAL: Rollback failed after transaction write error: {rollback_err}")

        if attempt < MAX_RETRIES - 1:
            time.sleep(1)  # Brief pause before retry
        continue

# All retries exhausted -- log the failure
logger.error(f"Transaction failed after {MAX_RETRIES} attempts for card {normalized_card}: {last_exception}")
return jsonify({'error': 'Transaction failed -- please try again'}), 503
```

**Key details:**
- The `trans_sheet` and `timestamp` are computed inside the loop because each retry needs a fresh timestamp.
- Email sending is OUTSIDE the retry scope (after successful return) and wrapped in its own try/catch so email failures don't trigger transaction rollback.
- The rollback restores `current_balance` (the original value before deduction).
- If rollback itself fails, log a CRITICAL-level message with all context for manual recovery.
- Only log the final failure, not intermediate attempts (per user decision).

**Ensure `import time` is at the top of the file.** If not present, add it.

**Ensure logger is available.** Add `import logging; logger = logging.getLogger(__name__)` at the top if not already present. Or use `get_logger` if the pattern from errors.py is used in this file.
