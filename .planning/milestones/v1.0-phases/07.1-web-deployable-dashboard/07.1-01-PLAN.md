Specifically remove any emits inside `register_student`, `report_lost_card`, or similar card-scan confirmation routes. These served the desktop card-scan UI which is hidden when `arduino_available=False`.

**CHANGE — hardware detection:**
4. In the `dashboard()` route, change `arduino_available=True` to `arduino_available=False`. Comment: `# web_app.py: hardware not expected; cashier detects per-session`.
5. In any other route that passes `arduino_available`, set it to `False`.

**ADD — JWT_SECRET startup guard (per CONTEXT.md locked decision):**
6. After the existing `FLASK_SECRET_KEY` startup guard block, add:
   ```python
   # --- JWT_SECRET startup guard ---
   _jwt_secret = os.getenv('JWT_SECRET', '').strip()
   _JWT_INSECURE_DEFAULT = 'bangko-jwt-secret-2026'
   if not _jwt_secret or _jwt_secret == _JWT_INSECURE_DEFAULT:
       logger.critical("event=startup_aborted reason=insecure_jwt_secret message=\"JWT_SECRET is not set or is using the insecure default. Set a strong random key in your .env file.\"")
       sys.exit(1)
   ```

**KEEP — module-level exports:**
7. Ensure `get_sheets_client` and `get_philippines_time` remain as module-level functions (they already are — do not move them).

10. Keep all non-hardware blueprint registrations (cashier_bp, etc.).

**Note on SocketIO:** Keep Flask-SocketIO in web_app.py (for local/desktop use). The WSGI mode on PythonAnywhere will receive WebSocket upgrade requests that are silently rejected — this is acceptable behavior and does not crash the app. Only remove `emit()` calls from regular HTTP route handlers (not from `@socketio.on` handlers).
