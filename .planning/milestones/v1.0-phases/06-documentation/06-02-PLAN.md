**Structure:**

**H1: API Reference**

**H2: Overview**
- Base URL: `http://<server-ip>:5001` (port from API_PORT env var, default 5001)
- Content-Type: `application/json` for all requests and responses
- Two auth mechanisms — explain the distinction clearly:
  - **Session Token**: Returned by `POST /api/auth/login`. Used by student endpoints. Header: `Authorization: Bearer <session_token>`. Validated by dict lookup (`active_sessions`), NOT JWT decode.
  - **JWT Token**: HS256-signed token. Used by cashier/admin endpoints marked with `@require_auth`. Header: `Authorization: Bearer <jwt_token>`. Generated by admin dashboard, not by this API directly.
  - **Device Token (NFC only)**: Additional header `X-Device-Token` required for `POST /api/nfc/pay`.

**H2: Endpoint Index**
Markdown table: Method | Path | Auth | Description — listing all 12 endpoints.

**H2: Endpoints** (one H3 per endpoint)

For EACH of the 12 endpoints, write:
- H3: `METHOD /path/to/endpoint`
- Description: one sentence
- Auth: "None", "Session Token (Bearer)", "JWT (Bearer)", or "JWT (Bearer) + X-Device-Token"
- Request: table of params (query params, body fields, headers) with Name | Type | Required | Description
- Response 200: JSON example with realistic values
- Response error cases: table with Status | Condition

**Endpoint 1: GET /api/health**
No auth. Returns `{"status": "healthy"}`.

**Endpoint 2: POST /api/auth/login**
Body: `{"student_id": "202501"}`. Returns session token + student object (see JSON in interfaces). Error: 401 if student not found or card status inactive.

**Endpoint 3: POST /api/auth/logout**
Session Bearer. Body: none. Removes token from active_sessions. Returns `{"message": "Logged out"}`.

**Endpoint 4: GET /api/student/profile**
Session Bearer. Returns student profile object from Users sheet.

**Endpoint 5: GET /api/student/balance**
Session Bearer. Returns `{"balance": 250.00}`. Balance from Money Accounts sheet col C.

**Endpoint 6: GET /api/student/transactions**
Session Bearer. Query: `limit` (default 20), `offset` (default 0). Returns paginated list from Transactions Log. Each item: timestamp, type, amount, balance_before, balance_after, items (JSON).

**Endpoint 7: POST /api/users/fcm-token**
Session Bearer. Body: `{"fcm_token": "..."}`. Stores FCM token in Users sheet FCMToken column. Returns `{"message": "FCM token registered"}`.

**Endpoint 8: GET /api/products**
No auth. Query: `category` (optional). Returns `{"products": [...]}`. Each product: id, name, category, price, image_url, active.

**Endpoint 9: POST /api/products**
JWT auth (admin/cashier role). Body: `{"name": "...", "price": 25.0, "category": "Food", "image_url": ""}`. Creates product in Products sheet. Returns 201 + created product.

**Endpoint 10: POST /api/cashier/transaction**
JWT auth. Body: `{"card_uid": "ABCD1234", "items": [{"name": "Rice", "price": 25.0, "quantity": 1}], "total": 25.0}`. Validates card UID format. Deducts balance. Writes Transactions Log (8-column format with BalanceBefore). Sends FCM notification if balance below threshold. Returns `{"success": true, "new_balance": 225.0}`.

**Endpoint 11: POST /api/nfc/register**
Session Bearer. Body: `{"device_token": "..."}`. Creates VirtualCard entry in VirtualCards sheet. Returns `{"virtual_card_token": "..."}`.

**Endpoint 12: POST /api/nfc/pay**
JWT (Bearer) + `X-Device-Token` header. Body: `{"virtual_card_token": "...", "items": [...], "total": 25.0}`. Validates virtual card token, deducts balance from linked Money Account. TransactionType='NFC Purchase'. Returns `{"success": true, "new_balance": 225.0}`.

**H2: Error Response Format**
```json
{"error": "description of what went wrong"}
```
Common HTTP status codes used: 200, 201, 400, 401, 403, 404, 500. Note that specific business error codes from `backend/errors.py` may appear in the error string.

**H2: Card UID Format**
8 hexadecimal characters, uppercase, no dashes: `ABCD1234`. Regex: `^[0-9A-Fa-f]{8}$`. Invalid UIDs are rejected at all entry points with a 400 response.

**H2: Troubleshooting**
- "401 on /api/auth/login" → check student_id exists in Users sheet, status is Active
- "401 on student endpoints" → session token expired or invalid; call /api/auth/login again
- "403 on cashier endpoints" → JWT token does not have admin/cashier role; use cashier dashboard login flow
- "400 card_uid invalid" → must be exactly 8 hex characters (e.g., ABCD1234, not ABCD-1234)
- "500 on transaction" → Google Sheets API error; check credentials.json and network

Cross-reference: [Google Sheets Schema](google-sheets-schema.md), [Architecture](architecture.md).

**Structure:**

**H1: Google Sheets Schema**

**H2: Overview**
BankongSeton uses Google Sheets as its database. The spreadsheet contains 7 sheets (tabs). 5 are created manually during setup; 2 (Products, VirtualCards) are auto-created on first use. Cross-reference to [Setup Guide](setup.md) for how to create the spreadsheet and service account.

**H2: Sheets** (one H3 per sheet)

**H3: Users**
Columns (8): StudentID, Name, IDCardNumber, MoneyCardNumber, Status, ParentEmail, DateRegistered, FCMToken
- FCMToken: added in Phase 4 (low-balance notifications). Must be included when creating the sheet manually. May be empty for students who haven't logged in to the app.
- Status: "Active" or "Inactive". Lost card reports may change this.
- IDCardNumber: the physical student ID card UID (RFID hex). MoneyCardNumber: the canteen money card UID (RFID hex).
- Who writes: api_server.py (FCM token update), admin_dashboard.py (all student management)

**H3: Money Accounts**
Columns (5): MoneyCardNumber, StudentIDCard, Balance, Status, LastUpdated
- Balance: float (Thai Baht). IMPORTANT: code uses `update_cell(row, 3, balance)` — Balance is hardcoded as column 3 (index 1-based). Do NOT reorder columns.
- Status: "Active" or "Inactive"
- Who writes: api_server.py, cashier_routes.py (balance deductions), admin_dashboard.py (load balance)

**H3: Transactions Log**
IMPORTANT: Two different write paths produce rows with different column layouts. The sheet should have 8 columns to accommodate the fuller format.

Canonical 8-column schema: Timestamp, MoneyCardNumber, TransactionType, Amount, BalanceBefore, BalanceAfter, Status, ItemsJson

Write paths:
1. **api_server.py (POST /api/cashier/transaction)**: 8 columns — includes BalanceBefore at position 5
2. **cashier_routes.py (POST /cashier/api/process-sale flow)**: 7 columns — NO BalanceBefore; cols are: Timestamp, MoneyCardNumber, TransactionType, Amount, BalanceAfter, Status, ItemsJson
3. **nfc_payments.py (POST /api/nfc/pay)**: Verify column count during writing

Note: Rows written by cashier_routes.py will have empty BalanceBefore (or BalanceAfter is shifted). This is a known inconsistency. When reading, check the source that wrote the row before assuming column positions.

- TransactionType values: 'Purchase' (cashier/API), 'NFC Purchase' (NFC pay endpoint)
- Amount: negative value for debits (e.g., -25.0 for a 25 Baht purchase)
- ItemsJson: JSON array of purchased items e.g., `[{"name": "Rice", "price": 25.0, "quantity": 1}]`

**H3: Lost Card Reports**

**H3: Products**
Columns (7): ID, Name, Category, Price, ImageURL, Active, DateAdded
- Auto-created by admin_dashboard.py `ensure_products_sheet()` on first product management page visit
- Active: "TRUE" or "FALSE" string (Google Sheets boolean). Only Active=TRUE products appear in cashier POS
- Category: used to group products in cashier grid (e.g., "Food", "Drinks", "Snacks")
- Who writes: admin_dashboard.py (CRUD), api_server.py (GET /api/products reads this sheet)

**H3: VirtualCards**
Columns (6): StudentID, VirtualCardToken, DeviceToken, MoneyCardNumber, CreatedAt, IsActive
- Auto-created by nfc_payments.py on first NFC registration
- VirtualCardToken: UUID or similar unique token generated on registration
- DeviceToken: device-specific token for X-Device-Token header authentication
- MoneyCardNumber: links to Money Accounts sheet for balance lookup
- IsActive: "TRUE" or "FALSE"

**H3: Settings**
Columns (2): Key, Value
- Used to store global configuration
- Currently used for: `low_balance_threshold` (float, default 50.0)
- Read by api_server.py during each cashier transaction to decide whether to send FCM notification
- Admin can update via admin dashboard settings page or by editing the sheet directly

**H2: Relationships**
Diagram or description: Users.MoneyCardNumber → Money Accounts.MoneyCardNumber; Users.StudentID → VirtualCards.StudentID; Money Accounts.MoneyCardNumber → Transactions Log.MoneyCardNumber; VirtualCards.MoneyCardNumber → Money Accounts.MoneyCardNumber

**H2: Card UID Format**
All card UIDs (IDCardNumber, MoneyCardNumber, VirtualCardToken) are validated at API entry points. Physical RFID UIDs: 8 hex chars uppercase, regex `^[0-9A-Fa-f]{8}$`.

**H2: Troubleshooting**
- "Balance not updating after transaction" → check Money Accounts column C is Balance (never reorder columns)
- "FCM notifications not sending" → check FCMToken column exists in Users sheet (col 8), check Settings sheet has low_balance_threshold row
- "Products not appearing in cashier POS" → check Active column is 'TRUE' (not 'true' or '1'), check Products sheet exists
- "VirtualCard registration fails" → VirtualCards sheet is auto-created but requires service account has Edit access on spreadsheet
