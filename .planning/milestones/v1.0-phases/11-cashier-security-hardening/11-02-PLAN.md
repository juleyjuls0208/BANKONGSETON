<objective>
Remove the insecure hardcoded fallback from `JWT_SECRET` in `cashier_routes.py`
(`os.getenv("JWT_SECRET", "bangko-jwt-secret-2026")`), and add a matching startup
guard to `admin_dashboard.py` (which `web_app.py` already has but `admin_dashboard.py`
does not).

Purpose: Eliminates the SEC-02 vulnerability — a known default JWT signing secret
committed in source code. Any JWT signed with the default would be accepted by the
running service, making token forgery trivially easy.

Output:
- `cashier_routes.py` module-level `JWT_SECRET` line replaced; secret read inside
  the `record_once` callback (which already exists after Plan 11-01)
- `admin_dashboard.py` gets a JWT_SECRET guard block matching `web_app.py` lines 70-75
- `.env.example` gains a JWT_SECRET section with generation instructions
- `.env` gains a JWT_SECRET entry
</objective>
