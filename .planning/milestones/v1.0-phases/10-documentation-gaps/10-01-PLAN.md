```markdown
---

## Cashier Blueprint API

> **Note:** These endpoints are served by the **cashier dashboard server** (a separate Flask app, default port `5003`), NOT by `api_server.py` (port `5001`). They are NOT accessible via the student REST API base URL.

### Authentication

Cashier endpoints use a JWT stored in an **HttpOnly cookie** named `jwt_token`, set by `POST /cashier/api/login`. This is different from the Bearer token in the `Authorization` header used by `api_server.py`.

Protected routes verify the cookie with a custom `@jwt_required` decorator. Routes without this decorator require no authentication.

### Cashier Endpoint Index

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/cashier/login` | None | Renders cashier login page |
| POST | `/cashier/api/login` | None | Authenticates and sets JWT cookie |
| GET | `/cashier/` | JWT cookie | Renders cashier POS page |
| GET | `/cashier/api/ports` | JWT cookie | Lists available COM ports |
| POST | `/cashier/api/connect-arduino` | JWT cookie | Opens Arduino serial connection |
| GET | `/cashier/api/products` | JWT cookie | Returns all products |
| POST | `/cashier/api/logout` | None | Clears JWT cookie |
| GET | `/cashier/api/lookup-student` | JWT cookie | Searches students by name or ID |
| POST | `/cashier/api/process-sale` | JWT cookie | Initiates a sale transaction |
| POST | `/cashier/api/complete-sale` | JWT cookie | Finalises sale, deducts balance, sends FCM |

---

### GET /cashier/login

**Auth:** None

Renders the cashier login HTML page.

**Response 200:** HTML page

---

### POST /cashier/api/login

**Auth:** None

Authenticates the cashier with username and password. On success, sets a `jwt_token` HttpOnly cookie valid for 8 hours.

**Request Body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| username | string | Yes | Cashier username |
| password | string | Yes | Cashier password |

**Response 200:**
```json
{ "success": true }
```
Sets `jwt_token` HttpOnly cookie (8-hour expiry).

**Response 401:**
```json
{ "error": "Invalid credentials" }
```

---

### GET /cashier/

**Auth:** JWT cookie required

Renders the cashier POS HTML page. Redirects to `/cashier/login` if the JWT cookie is absent or invalid.

**Response 200:** HTML page

---

### GET /cashier/api/ports

**Auth:** JWT cookie required

Returns a list of available serial COM ports for Arduino connection. Returns an empty list when pyserial is unavailable (web/cloud mode).

**Response 200:**
```json
{ "ports": ["COM3", "COM4"] }
```

Returns `{"ports": []}` in web/cloud deployments where pyserial is not installed.

---

### POST /cashier/api/connect-arduino

**Auth:** JWT cookie required

Opens a serial connection to the specified COM port.

**Request Body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| port | string | Yes | COM port name, e.g. `"COM3"` |

**Response 200:**
```json
{ "success": true, "message": "Connected to COM3" }
```

**Error Responses:**

| Status | Condition |
|--------|-----------|
| 400 | `port` field missing from request |
| 500 | Connection failed (reason included in message) |
| 500 | Arduino bridge not available |
| 503 | Service unavailable (transient error) |

---

### GET /cashier/api/products

**Auth:** JWT cookie required

Returns all products including inactive ones. The `active` boolean field is included so the POS frontend can filter.

**Response 200:**
```json
{
  "products": [
    { "id": "PROD-001", "name": "Fried Rice", "category": "Food", "price": 25.0, "active": true }
  ]
}
```

**Response 503:**
```json
{ "error": "Service unavailable, please try again" }
```

---

### POST /cashier/api/logout

**Auth:** None (always succeeds)

Clears the `jwt_token` cookie regardless of whether it exists.

**Response 200:**
```json
{ "success": true }
```

---

### GET /cashier/api/lookup-student

**Auth:** JWT cookie required

Searches students by name or ID for use in manual (web) payment mode.

**Query Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| q | string | Yes | Search string (minimum 2 characters) |

**Response 200:**
```json
{
  "students": [
    { "id": "202501", "name": "Juan dela Cruz", "balance": 250.0, "card_uid": "ABCD1234" }
  ]
}
```

Returns an empty list if `q` is less than 2 characters. Returns up to 10 matches.

**Error Responses:**

| Status | Condition |
|--------|-----------|
| 503 | Google Sheets unavailable |
| 500 | Internal lookup error |

---

### POST /cashier/api/process-sale

**Auth:** JWT cookie required

Stores a pending transaction in the server session and emits a `cashier_request_card` WebSocket event to prompt card tap.

**Request Body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| items | array | Yes | List of `{name, price, qty}` objects |
| total | number | Yes | Total sale amount |

**Request Example:**
```json
{ "items": [{"name": "Rice", "price": 20, "qty": 1}], "total": 20.0 }
```

**Response 200:**
```json
{ "status": "waiting_for_card", "message": "Please tap student card" }
```
**Side effect:** Emits WebSocket event `cashier_request_card` with `{"timeout": 5000, "total": <total>}`.

**Response 400:**
```json
{ "error": "Invalid sale data" }
```

---

### POST /cashier/api/complete-sale

**Auth:** JWT cookie required

Finalises a pending transaction. Accepts either an RFID card UID (physical terminal) or a manual student ID (web mode). Deducts the balance, writes the transaction row to the Transactions Log sheet, sends FCM push notifications (fire-and-forget), and sends an email receipt if configured.

**Request Body (RFID card mode):**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| card_uid | string | Yes | 8-character hex RFID UID |

**Request Body (manual web mode):**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| manual_student_id | string | Yes | Student ID number |

**Response 200:**
```json
{ "success": true, "new_balance": 200.0, "timestamp": "2026-02-28 14:32:00" }
```

**Error Responses:**

| Status | Condition |
|--------|-----------|
| 400 | Neither `card_uid` nor `manual_student_id` provided |
| 400 | `card_uid` format invalid (must match `^[0-9A-Fa-f]{8}$`) |
| 400 | No pending transaction in session |
| 400 | Insufficient funds (`{"error": "Insufficient funds", "balance": ..., "required": ...}`) |
| 400 | Student has no card assigned (manual mode) |
| 403 | Card reported as lost |
| 403 | Card status not active |
| 404 | Card UID not found in Money Accounts sheet |
| 404 | Student not found (manual mode) |
| 500 | Manual student lookup failed |
| 503 | Google Sheets unavailable |
```

### Push Notifications

After a sale is completed, the cashier POS **does** send FCM push notifications to the student's registered Android device:

- **Purchase notification** — always sent after a successful `complete-sale` call.
- **Low-balance notification** — additionally sent if the student's new balance is below the configured `LOW_BALANCE_THRESHOLD` (default: ₱50).

Both notifications are **fire-and-forget**: a failure to deliver the push notification never blocks the transaction or triggers a rollback. If the student has not registered an FCM token (i.e., has never logged into the student app), no notification is sent — this is expected and silent.

> **Note:** FCM requires the student's `FCMToken` to be populated in the Users sheet. Tokens are registered automatically when a student logs into the Android app for the first time.
```
