    ### Section 3: API Reference

    **3.1 POST /api/nfc/register**
    - Base URL: backend API server (same server as `/api/student/*` endpoints)
    - Authentication: `Authorization: Bearer <session_token>` — use the student login session token from `POST /api/auth/login`, NOT a JWT
    - Request body: none required (student identity comes from session)
    - Success response 200: exact JSON shape from the working code
    - Error table:

    | HTTP Status | Error JSON | Cause | Resolution |
    |-------------|-----------|-------|-----------|
    | 401 | `{"error": "Invalid or expired token"}` | Session token missing, expired, or not found | Re-login via `/api/auth/login` |
    | 403 | `{"error": "No money card registered"}` | Student has no RFID card linked in Money Accounts sheet | Admin must register RFID card for this student |
    | 503 | `{"error": "Service unavailable, please try again"}` | Google Sheets API unreachable | Retry with exponential backoff |

    **3.2 POST /api/nfc/pay**
    - Authentication: Two headers required:
      1. `Authorization: Bearer <cashier_jwt_token>` — cashier's JWT (role: cashier or admin)
      2. `X-Device-Token: <device_token>` — the device_token received during registration, extracted from the NFC payload alongside the virtual_card_token
    - Request body: exact fields from working code (virtual_card_token, items array, total)
    - Success response 200: exact JSON shape
    - Error table:

    | HTTP Status | Error JSON | Cause | Resolution |
    |-------------|-----------|-------|-----------|
    | 400 | `{"error": "Invalid transaction data"}` | Missing virtual_card_token, items, or total ≤ 0 | Check request body format |
    | 401 | `{"error": "No token provided"}` | Missing Authorization header | Send cashier JWT |
    | 401 | `{"error": "Invalid or expired token"}` | Bad or expired cashier JWT | Re-authenticate cashier |
    | 401 | `{"error": "X-Device-Token header required"}` | Missing X-Device-Token header | Include X-Device-Token in request |
    | 401 | `{"error": "Invalid virtual card token or device token"}` | Token mismatch, inactive card, or wrong device token | Student must re-register; card may have been replaced |
    | 400 | `{"error": "Insufficient funds", "balance": <float>}` | Student balance below total | Show balance to student |
    | 503 | `{"error": "Service unavailable, please try again"}` | Google Sheets API unreachable | Retry with exponential backoff |

    **4.2 BankoHceService.kt** — copy-paste ready Kotlin:
    ```kotlin
    package com.example.bankongseton.nfc

    import android.nfc.cardemulation.HostApduService
    import android.os.Bundle
    import android.security.keystore.KeyGenParameterSpec
    import android.security.keystore.KeyProperties
    import androidx.security.crypto.EncryptedSharedPreferences
    import androidx.security.crypto.MasterKeys

    class BankoHceService : HostApduService() {
        private val SELECT_AID_HEADER = byteArrayOf(0x00, 0xA4.toByte(), 0x04, 0x00)
        private val RESPONSE_OK = byteArrayOf(0x90.toByte(), 0x00)
        private val RESPONSE_UNKNOWN = byteArrayOf(0x6D, 0x00)
        private val AID = byteArrayOf(
            0xF0.toByte(), 0x49, 0x49, 0x4F, 0x4E, 0x41
        )

        override fun processCommandApdu(apdu: ByteArray, extras: Bundle?): ByteArray {
            // Respond to SELECT AID command
            if (apdu.size >= 6 && apdu.sliceArray(0..3).contentEquals(SELECT_AID_HEADER)) {
                val tokenAndDeviceToken = getStoredTokens() ?: return RESPONSE_UNKNOWN
                // NFC payload: "virtual_card_token|device_token" (pipe-delimited)
                // Cashier software splits on '|' to get both values
                val payloadBytes = tokenAndDeviceToken.toByteArray(Charsets.UTF_8)
                return payloadBytes + RESPONSE_OK
            }
            return RESPONSE_UNKNOWN
        }

        override fun onDeactivated(reason: Int) { /* no-op */ }

        /**
         * Returns "virtual_card_token|device_token" from EncryptedSharedPreferences.
         * Returns null if not registered.
         */
        private fun getStoredTokens(): String? {
            val prefs = EncryptedSharedPreferences.create(
                "nfc_prefs",
                MasterKeys.getOrCreate(MasterKeys.AES256_GCM_SPEC),
                applicationContext,
                EncryptedSharedPreferences.PrefKeyEncryptionScheme.AES256_SIV,
                EncryptedSharedPreferences.PrefValueEncryptionScheme.AES256_GCM
            )
            val token = prefs.getString("virtual_card_token", null) ?: return null
            val deviceToken = prefs.getString("device_token", null) ?: return null
            return "$token|$deviceToken"
        }
    }
    ```

    **4.3 Registration API call (Kotlin)** — call after student login:
    ```kotlin
    // Call after successful login, using the session token
    suspend fun registerNfcCard(sessionToken: String): NfcRegistrationResult {
        val response = apiService.registerNfcCard(
            authorization = "Bearer $sessionToken"
        )
        if (response.isSuccessful) {
            val body = response.body()!!
            // Store both tokens securely
            val prefs = EncryptedSharedPreferences.create(
                "nfc_prefs",
                MasterKeys.getOrCreate(MasterKeys.AES256_GCM_SPEC),
                context,
                EncryptedSharedPreferences.PrefKeyEncryptionScheme.AES256_SIV,
                EncryptedSharedPreferences.PrefValueEncryptionScheme.AES256_GCM
            )
            prefs.edit()
                .putString("virtual_card_token", body.virtualCardToken)
                .putString("device_token", body.deviceToken)
                .apply()
        }
        return response
    }

    // Retrofit interface
    interface BankoApiService {
        @POST("/api/nfc/register")
        suspend fun registerNfcCard(
            @Header("Authorization") authorization: String
        ): Response<NfcRegistrationResponse>
    }

    data class NfcRegistrationResponse(
        @SerializedName("virtual_card_token") val virtualCardToken: String,
        @SerializedName("device_token") val deviceToken: String,
        @SerializedName("money_card") val moneyCard: String,
        val message: String
    )
    ```

    **4.4 Payment flow (Cashier POS → Backend)**
    The cashier NFC reader reads the phone's NFC payload: `"virtual_card_token|device_token"` (pipe-delimited string). The cashier POS software:
    1. Splits the payload on `|` to get `virtual_card_token` and `device_token`
    2. Sends `POST /api/nfc/pay` with:
       - Header: `Authorization: Bearer <cashier_jwt>`
       - Header: `X-Device-Token: <device_token>` (second part of payload)
       - Body: `{ "virtual_card_token": "<first part>", "items": [...], "total": <float> }`

    ### Section 6: Important Notes for v2 Implementation
    - Token has no expiry — valid until student re-registers. Re-registration silently deactivates the old row.
    - If `401 Invalid virtual card token or device token` after a valid registration, the student has re-registered and the Android app must call `/api/nfc/register` again to get fresh tokens.

    - The cashier hardware must support ISO 14443-4 (HCE) in addition to the existing RFID (ISO 14443-3) reads.
