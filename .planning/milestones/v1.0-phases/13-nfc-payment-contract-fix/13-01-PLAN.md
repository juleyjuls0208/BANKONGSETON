<objective>
Fix the NFC-03 field name mismatch: the Android `NfcRegistrationResponse` data class declares `virtual_token` but the Python backend already returns `virtual_card_token`. Retrofit silently maps the wrong field name, causing the stored token to be null/empty and breaking all subsequent NFC payments.

Purpose: The backend contract is already correct (nfc_payments.py:127, api_server.py:652 both return `virtual_card_token`). This plan fixes the Android side to match.
Output: `ApiClient.kt` and `NfcManager.kt` updated so the token field name is consistent end-to-end.
</objective>
