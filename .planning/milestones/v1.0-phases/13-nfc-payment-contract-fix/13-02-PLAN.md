<objective>
Fix the NFC-04 contract mismatch: the Android HCE service (`BankoHceService`) can only supply `virtual_card_token` to the payment terminal — it has no way to obtain or send the `X-Device-Token` header. The current `/api/nfc/pay` endpoint requires both, causing all NFC payments to fail with 401.

Purpose: The payment terminal (acting as the Retrofit client) calls `/api/nfc/pay` directly using only the card token embedded in the APDU response. The server must look up the device token internally.
Output: A new `NFCService.get_virtual_card_by_token()` method + updated `nfc_pay()` handler that removes the `X-Device-Token` requirement.
</objective>

def get_virtual_card_by_tokens(
    self,
    virtual_card_token: str,
    device_token: str,
    db,
) -> Optional[dict]:
    """Return the active VirtualCard row matching both tokens, or None."""
    vc_sheet = ensure_virtual_cards_sheet(db)
    records = vc_sheet.get_all_records()
    for r in records:
        if (
            r.get('VirtualCardToken') == virtual_card_token
            and r.get('DeviceToken') == device_token
            and str(r.get('IsActive', '')).upper() == 'TRUE'
        ):
            return r
    return None

        return None
