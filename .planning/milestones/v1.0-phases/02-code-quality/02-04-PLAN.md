**Import setup for each file:**
Add at the top of each file (near other imports):
```python
from errors import get_logger

```

**Log level mapping rules:**
- Startup messages → logger.info("event=startup ...")
- Card UID read / card scan events → logger.info("event=card_read uid=%s", uid)
- Google Sheets API calls → logger.info("event=sheets_query sheet=%s", sheet_name)
- Auth success / failure → logger.info("event=auth_success") / logger.warning("event=auth_failed reason=...")
- Errors (exceptions, API failures) → logger.error("event=... error=%s", str(e))
- Debug/verbose output → logger.debug("event=... detail=%s", ...)
- Serial/Arduino connection messages → logger.info("event=serial_connect port=%s", port)

**Key=value format:**
- Every log message MUST start with `event=<snake_case_name>`
- Additional context as key=value pairs: `uid=%s student_id=%s sheet=%s`
- No free-form sentences — structured pairs only
- Example: `logger.info("event=card_read uid=%s normalized=%s", raw_uid, normalized_uid)`

**admin_dashboard.py has ~60 print() calls.** Work through the file systematically, section by section. The logger variable should be module-level (after imports) not re-instantiated inside each function.

**api_server.py has ~5 print() calls.** Same approach.

AVOID: logger.info("Card UID read: " + uid) — this is not key=value format.
USE: logger.info("event=card_read uid=%s", uid) — structured, lazy-formatted (% not .format()).
AVOID: Changing any business logic while replacing prints — only the logging calls change.
AVOID: Using print() for any purpose, including debugging — use logger.debug().
