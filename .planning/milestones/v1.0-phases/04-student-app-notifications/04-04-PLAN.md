**SecureStorage.kt — add saveLastBalance / getLastBalance:**

Add after the `isLoggedIn()` function:
```kotlin
fun saveLastBalance(balance: Double) {
    sharedPreferences.edit().putFloat(KEY_LAST_BALANCE, balance.toFloat()).apply()
}

fun getLastBalance(): Double? {
    val stored = sharedPreferences.getFloat(KEY_LAST_BALANCE, -1f)
    return if (stored >= 0f) stored.toDouble() else null
}
```

Add to companion object:
```kotlin
private const val KEY_LAST_BALANCE = "last_balance"
```

**LoginActivity.kt — register FCM token after successful login:**

In `performLogin()`, inside the `if (response.isSuccessful)` block, AFTER saving auth data and BEFORE starting HomeActivity, add FCM token registration:

```kotlin
// Register FCM token if available (buffered from FCMService.onNewToken)
val fcmToken = getSharedPreferences("bangko_prefs", Context.MODE_PRIVATE)
    .getString("fcm_token", null)
if (!fcmToken.isNullOrEmpty()) {
    ApiClient.apiService.registerFCMToken(
        "Bearer ${loginResponse.token}",
        FCMTokenRequest(fcmToken)
    ).enqueue(object : Callback<MessageResponse> {
        override fun onResponse(call: Call<MessageResponse>, response: Response<MessageResponse>) {
            // Silent success — token registration is best-effort
        }
        override fun onFailure(call: Call<MessageResponse>, t: Throwable) {
            // Silent failure — app continues without push notifications
        }
    })
}
```

Required imports to add to LoginActivity.kt:
- `android.content.Context` (may already be imported via other usage)
- The `Callback`, `Response`, `Call` imports are already present

Note: FCM token registration is fire-and-forget. The login flow completes immediately — FCM registration happens asynchronously in the background and its failure is silent.
